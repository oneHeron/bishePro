# 社区检测集成平台（毕业设计）- Codex 工作指令

## 总目标
做一个开源平台：集成社区检测方法 + 常用数据集 + 常用评价指标。
未登录：只能浏览方法/数据集/指标；登录后：可发起运行并查看结果与历史。

## 技术栈
- 前端：Vue3 + Vite（UI 适当美观、人性化）
- 后端：Python（建议 FastAPI）
- 算法/指标：Python
- 环境：默认同一 conda env（python=3.8）(采用miniconda，新建环境名为bsenv)
- 运行：异步任务（提交 run -> 返回 run_id -> 查询状态/结果）

## 初始内置
数据集：Karate、Dolphin、Polbooks（仅拓扑），Cora、Citeseer（含标签/特征）
指标：NMI、ACC、ARI、模块度 Q
方法：DeepWalk、K-Means、Louvain、NMF
必须预留“快捷加入方法/数据集/指标”的接口（插件化/注册机制）

## 架构建议（必须遵守接口契约）
- 后端分层：API 层 / 服务层 / 运行器 runner / 插件 plugins / 数据集 datasets / 指标 metrics
- 运行必须异步：提交 run -> 返回 run_id -> 前端轮询或 WS 获取状态 -> 展示结果
- 运行记录要落库：run 的参数、日志、指标结果、耗时、随机种子、版本信息

## 可扩展性要求（必须实现）
- 方法/数据集/指标都走“注册/插件化”机制（新增时不改核心流程）
- 数据集支持：本地提供 + 可选网络下载缓存
- 方法支持：CPU 默认；部分方法可标记 requires_gpu，并进入 GPU 队列

## 运行与记录
- 每次运行必须记录：参数、随机种子、耗时、日志、版本信息、指标结果；对于数据集节点小于100的数据集，做一个拓扑结构的可视化展示。
- API 必须返回清晰错误信息（比如缺失标签无法算 ACC/ARI 等）

## 安全与可控性
- 禁止执行危险命令（rm -rf、关机、修改系统配置等）
- 任何需要网络/系统级权限的操作必须先询问确认
- 依赖新增要先说明原因、影响与替代方案

## 交付标准
- 能完整跑通：选择(方法+数据集+指标) -> 异步运行 -> 返回结果并展示
- 提供最小可用 UI：列表页、登录注册页、运行页、运行历史页、结果详情页
- 提供 README：本地启动方式、如何添加新方法/数据集/指标
- 提供基础测试：至少覆盖插件注册/数据集加载/指标计算/一次完整 run 的 API 测试
