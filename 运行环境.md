# 运行环境说明

本文档说明社区检测集成平台的本地运行环境、依赖安装和启动命令。

## 1. 基础环境

推荐使用 Linux / WSL / macOS 运行。

需要提前安装：

- Miniconda / Anaconda
- Node.js 18+ 与 npm
- 可选：Redis，用于 RQ 异步队列
- 可选：Docker 与 Docker Compose，用于容器化启动

项目默认 Python 环境：

- Python：3.8
- Conda 环境名：`bsenv`

## 2. 创建并激活 Python 虚拟环境

在项目根目录 `/home/bishePro` 下执行：

```bash
conda create -y -n bsenv python=3.8
conda activate bsenv
```

如果已经创建过环境，只需要激活：

```bash
conda activate bsenv
```

## 3. 安装后端依赖

后端依赖文件位于 `backend/requirements.txt`。

```bash
cd /home/bishePro/backend
pip install -r requirements.txt
```

当前后端基础依赖包括：

- `fastapi`
- `uvicorn`
- `pydantic`
- `SQLAlchemy`
- `PyJWT`
- `pytest`
- `httpx`
- `rq`
- `redis`
- `typing_extensions`

## 4. 安装前端依赖

前端位于 `frontend/`，技术栈为 Vue3 + Vite。

```bash
cd /home/bishePro/frontend
npm install
```

当前前端主要依赖包括：

- `vue`
- `vue-router`
- `vite`
- `@vitejs/plugin-vue`

## 5. 可选算法依赖

基础平台、内置接口和普通运行可先只安装后端与前端依赖。

如果需要运行 CSEA 方法，可安装其额外依赖：

```bash
cd /home/bishePro/core_modules/methods/csea
pip install -r requirements.txt
```

该文件包含：

- `pandas`
- `networkx`
- `numpy`
- `scikit-learn`
- `keras`
- `tensorflow-gpu`
- `scipy`

如果需要运行 CDBNE、DDGAE、GNN Template 等基于 PyTorch 的方法，还需要根据本机 CPU/GPU 和 CUDA 版本安装 PyTorch 相关包，例如：

```bash
pip install torch
```

若需要 GPU 版本，请到 PyTorch 官方安装页选择与本机 CUDA 匹配的安装命令。

## 6. 启动后端

开发模式启动 FastAPI：

```bash
cd /home/bishePro/backend
conda activate bsenv
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

启动后可访问：

- 后端 API：`http://127.0.0.1:8000`
- Swagger 文档：`http://127.0.0.1:8000/docs`

默认情况下，`RUNNER_BACKEND=auto`，系统会优先尝试 RQ + Redis，失败后回退到本地线程异步执行。

## 7. 启动前端

新开一个终端执行：

```bash
cd /home/bishePro/frontend
npm run dev
```

启动后访问：

```text
http://127.0.0.1:5173
```

前端默认请求后端地址：

```text
http://127.0.0.1:8000
```

## 8. 使用 Redis + RQ 异步队列

如果希望使用正式异步队列，需要先启动 Redis。

后端 API 服务：

```bash
cd /home/bishePro/backend
conda activate bsenv
export RUNNER_BACKEND=rq
export REDIS_URL=redis://127.0.0.1:6379/0
export RUN_QUEUE=runs
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

RQ Worker：

```bash
cd /home/bishePro/backend
conda activate bsenv
export REDIS_URL=redis://127.0.0.1:6379/0
export RUN_QUEUE=runs
python -m rq worker runs
```

如果使用 CPU / GPU 分流队列，可设置：

```bash
export RUN_QUEUE_CPU=cpu
export RUN_QUEUE_GPU=gpu
```

并分别启动 worker：

```bash
python -m rq worker cpu
python -m rq worker gpu
```

## 9. Docker Compose 一键启动

项目根目录提供 `docker-compose.yml`。

```bash
cd /home/bishePro
docker compose up --build
```

启动后：

- Backend：`http://127.0.0.1:8000`
- Redis：`127.0.0.1:6379`
- Worker：消费 `runs` 队列

停止：

```bash
docker compose down
```

## 12. 常用环境变量

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `RUNNER_BACKEND` | `auto` | 运行后端：`auto` / `rq` / `redis` / `inline` |
| `REDIS_URL` | `redis://127.0.0.1:6379/0` | Redis 连接地址 |
| `RUN_QUEUE` | `runs` | 默认 RQ 队列名 |
| `RUN_QUEUE_CPU` | 空 | CPU 队列名 |
| `RUN_QUEUE_GPU` | 空 | GPU 队列名 |
| `RUNNER_CONDA_ENV` | `bsenv` | 方法子进程使用的 conda 环境 |
| `RUNNER_CONDA_EXE` | 空 | conda 可执行文件路径 |
| `RUNNER_METHOD_TIMEOUT_SEC` | `7200` | 单次方法执行超时时间，单位秒 |
| `DATASETS_ROOT` | `/home/bishePro/datasets` | 数据集根目录 |

## 13. 常见问题

- 前端页面打不开：确认 `npm run dev` 是否正常运行，并访问 `http://127.0.0.1:5173`。
- 前端接口请求失败：确认后端是否运行在 `http://127.0.0.1:8000`。
- `401 Invalid token`：需要先登录，并携带 `Authorization: Bearer <token>`。
- 指标提示缺少标签：部分拓扑数据集没有真实标签，不能计算 NMI、ACC、ARI 等依赖标签的指标。
- Redis 队列不可用：可先不启动 Redis，使用默认 `RUNNER_BACKEND=auto` 回退到本地线程异步执行。
- GPU 方法无法运行：确认已安装对应 CUDA 版本的深度学习框架，并检查 `requires_gpu`、`use_gpu` 和队列配置。
